Destination Sync

          
Incremental 会关闭当前 source cycle，并且只同步当前选中的 destination。它不会额外设置 full rescan 标记，但传输前仍会比较当前 source 和 destination 快照。

          
Changed Since 会关闭当前 source cycle，完整扫描 source，并以上次该 destination verified 的 source cycle 时间作为起点，只同步从那之后变化过的相对路径。

          
Full 也只作用于当前选中的 destination，同时会把本轮 cycle 标记为需要完整重扫。它会做完整 source/destination reconcile，然后只复制缺失或不一致的文件。

        

        

          
Restart Recovery

          
进程重启后，两种模式都会比较当前 source 和当前 destination。目标端已经存在且匹配的文件会跳过；缺失文件、不一致文件、类型变化，以及未完成的临时传输都会被修复。

          
Full 不是删除后全部重传，而是完整校验加修复。开启 Mirror 时，destination 中多出来且 source 不存在的路径会被删除。

        

        

          
Example

          
对于 \\?\C:\Users\tiger\Documents\xwechat_files 到 nas:/opt，实际 destination root 通常是 /opt/xwechat_files。

          
如果重启前已经同步了一部分，Incremental 会继续补齐缺口并修复不一致；Changed Since 会完整扫描 source 但只处理上次 verified cycle 之后变化的路径；Full 会扫描整棵树、验证目标端、修复差异，并在 Mirror 开启时删除额外文件。
