Destination Sync

          
Incremental 会关闭当前 source cycle，并且只同步当前选中的 destination。对 Realtime 目标它应用积压的事件路径；ZFS 源上如果有已验证的基准快照，会走 zfs diff 快路径只同步差异。

          
Full 是完整对账加同步，修复包括目标侧漂移在内的所有差异；开启 Mirror 时删除 source 不存在的多余路径。有两个实现，按 ZFS diff 配置（Settings 或每个 destination 的 ⚙ 中可开关）自动选择：zfs diff 版——src 和 dst 都在本机 ZFS 且有已验证基准快照时，两侧各自与基准做 diff，只对账变化过的路径，秒级完成；对比版——源端和目标端并行扫描整棵树、对比清单、只传输缺失或不一致的文件。任一前提不满足自动落到对比版。

          
Scan（对账不同步）：生成差异报告（新增/更新/删除/类型变化），不改动任何文件。与 Full 相同的双版本策略：ZFS diff 可用时秒级，只比对两侧基准以来变化的路径；否则两端并行全树扫描。

        

        

          
可靠性行为

          
复制过程中源文件被修改不会把目标标红：这些路径记为黄色 source_changing 问题，下一轮自动收敛。单个文件失败不会中断整批传输（最多容忍 20 个），连接断开才会立即终止。

          
跨机传输的每个文件在落盘前都做 blake3 端到端校验，先写临时文件再原子改名，中断后可断点续传。

        

        

          
Restart Recovery

          
进程重启后会重新驱动未完成的 cycle：目标端已经存在且匹配的文件会跳过；缺失文件、不一致文件、类型变化，以及未完成的临时传输都会被修复。

          
注意：Realtime 目标的绿点表示"事件都已应用"，不代表整棵树被验证过。如果怀疑有漂移（比如进程停机期间的变更），先用 Scan 查看差异，再用 Full 对账修复。

        

        

          
Example

          
对于 \\?\C:\Users\tiger\Documents\xwechat_files 到 nas:/opt，实际 destination root 通常是 /opt/xwechat_files。

          
如果重启前已经同步了一部分，Incremental 会继续补齐缺口并修复不一致；Full 会对账并修复所有差异（ZFS diff 可用时秒级，否则并行扫描两端），并在 Mirror 开启时删除额外文件。
